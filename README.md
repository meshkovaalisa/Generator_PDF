# Document Template Processor

Веб-сервис на базе FastAPI для автоматической генерации PDF-документов. Приложение загружает данные об образовательных программах с внешнего API, распределяет их по факультетам, динамически заполняет ODP-шаблоны и конвертирует результат в PDF с помощью LibreOffice.

## Описание

Проект реализует полный цикл обработки документов: от получения сырых данных до выдачи готового PDF-файла. На старте сервер автоматически обращается к API университета, парсит JSON, разделяет программы по факультетам и сохраняет их в отдельные файлы. При запросе через веб-интерфейс или API система находит нужный факультет, очищает данные от HTML-разметки, распределяет их по уровням образования, подставляет значения в ODP-шаблон (используя синтаксис {{ключ}}), конвертирует презентацию в PDF и отдаёт файл пользователю. Все временные и промежуточные файлы автоматически удаляются при остановке сервера.

## Ключевые возможности

- Автоматическая загрузка и парсинг данных при инициализации приложения
- Группировка программ по факультетам с сохранением в JSON
- Очистка HTML-разметки и фильтрация полей в зависимости от уровня образования (бакалавриат/специалитет или магистратура/аспирантура)
- Динамическое дублирование слайдов в ODP-шаблонах под количество записей
- Замена плейсхолдеров вида {{key}} с удалением неиспользованных меток
- Конвертация ODP в PDF через headless-режим LibreOffice
- Простой веб-интерфейс для выбора факультета и скачивания документа
- REST API для интеграции с внешними системами
- Автоматическая очистка временных директорий при завершении работы сервера

## Требования

- Python версии 3.14 или выше
- Установленный pip
- Установленный LibreOffice (для конвертации документов)

## Установка и настройка

1. Клонируйте репозиторий:
git clone https://github.com/ваш-username/ваш-репозиторий.git
cd ваш-репозиторий

2. Создайте и активируйте виртуальное окружение:
python -m venv venv
source venv/bin/activate  # Для Linux/macOS
venv\Scripts\activate     # Для Windows

3. Установите зависимости из pyproject.toml:
pip install .

Для разработки с возможностью вносить изменения в код без переустановки используйте:
pip install -e .

4. Убедитесь, что LibreOffice установлен и путь к исполняемому файлу указан корректно в config.py (по умолчанию настроено для Windows).

## Конфигурация

Основные параметры вынесены в файл config.py:

- BASE_DIR: корневая директория проекта
- temp_dir: путь для временных файлов обработки
- rendered_files_dir: путь для сохранения готовых PDF
- faculties_dir: директория для JSON-файлов факультетов
- templates_dir: путь к ODP-шаблонам
- soffice_path: абсолютный путь к исполняемому файлу LibreOffice

При необходимости измените пути под вашу операционную систему. На Linux/macOS обычно достаточно указать soffice или /usr/bin/soffice.

## Запуск сервера

Для разработки с автоматической перезагрузкой при изменении кода:
python -m uvicorn main:app --reload --host 127.0.0.1 --port 8000

Для запуска без режима автоперезагрузки:
python -m uvicorn main:app --host 127.0.0.1 --port 8000

После запуска веб-интерфейс доступен по адресу: http://127.0.0.1:8000

## API Endpoints

| Метод | Путь | Описание |
|-------|------|----------|
| GET | / | Возвращает главную страницу с веб-интерфейсом |
| GET | /faculties | Возвращает список доступных факультетов в формате JSON |
| POST | /render | Генерирует и возвращает PDF для указанного факультета |

### Пример запроса POST /render

Тело запроса (application/json):
{
  "faculty_name": "Механико-математический факультет"
}

Ответ: файл PDF (application/pdf) или ошибка 404, если факультет не найден.

## Важные примечания

1. Зависимость от внешнего API: при запуске сервер синхронно загружает данные с https://mauniver.ru/api/get_programms.php. Если API недоступен, приложение завершит инициализацию с ошибкой 504 или 5xx.
2. LibreOffice обязателен: конвертация выполняется через CLI-вызов. Без установленного LibreOffice endpoint /render вернет ошибку subprocess.CalledProcessError.
3. Версия Python: проект требует Python >=3.14. Убедитесь, что виртуальное окружение создано для корректной версии интерпретатора.
4. Очистка данных: все временные и сгенерированные файлы автоматически удаляются при корректной остановке сервера благодаря механизму lifespan.
5. Шаблоны: ODP-файлы в static/templates/ должны использовать синтаксис {{ключ}} для подстановки данных. Имена файлов шаблонов должны совпадать с названиями факультетов.
